Granicus' Rapid Online Access Method (ROAM) is a robust, easy to use, enterprise data retrieval solution. ROAM allows you to run full text searches across all data elements, or selected data elements, with near instant results. Learn more about ROAM and its capabilities below


HELPFUL TIPS

BASIC SEARCH

  • The Basic Search is a full text search across all indexed fields.

    Example: If a user types John Smith into the Basic Search it will find all instances of John Smith located in all fields indexed.

ADVANCED SEARCH

  • The Advanced Search is a full text field-level search that can be accessed from the Advanced Search hyperlink on the main search page. The Advanced Search can be used to search by Case Number, Name, Date and more!

    Example: Select the Advanced Search hyperlink (shown above in Basic Search section)

    By selecting the Advanced Search hyperlink, ROAM will list all the field-level searches that can be performed (example below)

JUDICIAL CALENDAR

  • The judicial calendar displays a 180 day span based on the current date! 30 days in the past from todays date and 150 days in the future from todays date.

    To view a list of cases for a particular day, click on the linked numbers under the court room you wish to see.

HOW TO USE SEARCH FILTERS

  • ROAM provides the ability to filter searches utilizing ROAMs faceting functionality. The Facet is located on the left hand side of your search results screen. When selecting an item in the facet it will filter your results to include only results that match that item. Selected filters are shown at the top of the facet. Users have the option of removing filters by clicking on them again in the top of the facet.

HOW TO USE EXPORT OPTIONS

  • Once a search is performed with the basic search or advanced search, that search can be exported in three formats.
    Excel, PDF, or Word.
    Just select your prefered format and save it to a file.


SEARCH TIPS

  • AND | OR Search Option will return results according the search value provided by the user.

    AND is the more practical method of searching. Selecting this option will allow the user to search for multiple, consecutive words within fields and will limit the results displayed.

    Example selecting AND: If the user wants to search for the name "SMITH" and selects AND Case Type: "DIVORCE", the search will find "DIVORCE" records that contains the name "SMITH".

    OR is a versatile search method of searching.

    Example selecting OR: If a user wants to search for the name "SMITH J" and selects OR Case Type: "DIVORCE", the result search will be much more varied finding any record that contains the words"SMITH" OR "DIVORCE".

WILD CARD SEARCHING

  • ROAM supports Wildcard searching. A user can perform a Wildcard search by entering a * after a minimum of 2 characters in order to utilize wildcard searching capability.

    Example: Entering Jo* Smith in ROAM will return results matching John Smith, Smith John, Joe Smith, Smith Joe, Joseph Smith, Smith Joseph, etc. Entering J* Smith into ROAM will only return results matching J Smith and Smith J. You must enter a minimum of 2 characters to use ROAMs Wildcard searching capability

RESULTS PER PAGE

  • The results per page in ROAM can be adjusted by the user to any of the allowed settings in the results per page drop down located in the lower right hand corner of the web page.
BROWSER COMPATIBILITY




Browser Recommendation

ROAM strives to utilize the latest in web technology, so we recommend our users take advantage of the most current versions of available web browsers like Internet Explorer(IE) 10, Chrome 28, Firefox 22, etc., to get the best ROAM experience possible.

Internet Explorer Browser Mode and Document Mode: Like other browsers, Internet Explorer(IE) supports Browser and Document Modes. These modes affect the way webpages are interpreted and displayed. IE allows you to choose between support for the latest standard or support for certain behaviors popularized by older browsers.

As mentioned above, ROAM strives to bring users the latest in web technology and browser settings are a crucial part of displaying the ROAM web pages properly. Below is a quick walk through on how to check and change your IE browser and document mode settings which work best with ROAM.

  • Once you've launched your IE browser, press the F12 key on your keyboard. This will promp an action console to show at the bottom of the screen (Highlighted in green. See screen shot below). At the top of the console, there will be a menu bar with different menu options. From here, Browser Mode and Document Mode can be switched to support newer or older modes.

    For the best ROAM experience, we recommend using the latest modes, Browser Mode: IE9 and Document Mode:IE9 standards. To switch modes, simply hover over the menu item with your mouse and select the mode you wish to employ. Once selection has been made, press F12 to close the action console or click the "x" at the very right of the action console menu bar.

INSTRUCTIONS FOR DETERMINING PERFECTED SERVICE

Instructions for Determining Perfected Service via the Mobile County ROAM Public Access Site (Legacy Cases – filed prior to 08.01.2013)

  • Log in to Judicial ROAM Public Access
  • Click Case and Party Search tab
  • Execute Search
  • Click Case Number of Desired Case
  • Scroll to the Case Action Summary Grid
  • Sort the Comments column by clicking the column header one time
  • Scroll to the clustered, sorted events starting with Cause…
  • Scan through comments specific to Cause at issue for which a Service of Process inquiry is appropriate



Instructions for Determining Perfected Service via the Mobile County ROAM Public Access Site (Day Forward Cases – filed on or after 08.01.2013)
  • Log in to Judicial ROAM Public Access
  • Click Case and Party Search tab
  • Execute Search
  • Click Case Number of Desired Case
  • Scroll to the Service of Process Grid (located immediately below the Case Action Summary grid)
  • Sort the Executed column by clicking the column header one time
  • Review every record where the value in the Service of Process column is “No”
